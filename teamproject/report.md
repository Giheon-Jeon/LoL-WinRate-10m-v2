# League of Legends 초반 15분 데이터 기반 승패 예측 모델 개발

---

## 1. 프로젝트 개요

### 1.1 주제
League of Legends(이하 LoL) 경기에서 게임 시작 후 15분 시점의 데이터만으로 블루팀의 최종 승패를 예측하는 머신러닝 모델 개발

### 1.2 목표
- LoL 게임의 초반 15분 데이터를 활용하여 80% 이상의 정확도로 승패를 예측하는 모델 구축
- 승패에 영향을 미치는 핵심 게임 요소(라인별 골드 격차, 오브젝트, KDA 등)에 대한 정량적 인사이트 도출
- 단순 분류 모델을 넘어 게임 전략에 활용 가능한 해석 가능한 분석 제공

### 1.3 데이터 출처
- **Riot Games 공식 API**
  - 리그 API: `https://kr.api.riotgames.com`
  - 매치/타임라인 API: `https://asia.api.riotgames.com`
- **DataDragon API**: 챔피언 메타데이터 (버전 14.10.1, 167개 챔피언)

### 1.4 데이터 규모

| 항목 | 수치 |
|------|------|
| 수집 게임 수 | 8,410 경기 |
| 컬럼 수 | 80개 |
| 결측값 | 0개 |
| 블루팀 승 (win=1) | 4,127게임 (49.1%) |
| 블루팀 패 (win=0) | 4,283게임 (50.9%) |

## 2. 데이터 수집

### 2.1 수집 방법

#### 2.1.1 수집 파이프라인
1. **플레이어 PUUID 수집**: 티어별 리그 API로 상위 티어 플레이어 식별자 확보
2. **매치 ID 수집**: 각 플레이어의 최근 솔로랭크(queue=420) 매치 ID 10개씩 추출
3. **매치 상세 + 타임라인 동시 수집**: 매치 API와 타임라인 API 병행 호출

#### 2.1.2 15분 데이터 추출 방식
- 타임라인의 `frames[0:16]`을 순회하며 `timestamp ≤ 900,000ms`(15분) 이벤트만 집계
- 15분 시점의 라인별 골드/CS는 `frames[15].participantFrames`에서 직접 추출
- 챔피언 역할군은 DataDragon API로 사전 매핑 (Tank, Assassin, Mage, Fighter, Marksman, Support)

### 2.2 수집 대상 티어

상위 티어 게임만 수집하여 데이터 품질을 확보. 티어별 누적 매치 ID는 다음과 같다.

### 2.4 실패 원인 및 처리 방법

#### 2.4.1 실패 원인
1. **조기 종료 게임**: 타임라인 프레임이 15개 이하인 경우 (`len(frames) <= 15`) — 항복 등으로 15분 전에 끝난 게임
2. **API rate limit**: 분당 요청 한도 초과 시 429 응답
3. **4xx/5xx 응답**: 일부 매치의 데이터 누락 또는 서버 오류

#### 2.4.2 처리 방법
- **중간 저장**: 500게임 누적될 때마다 `lol_clean_{N}.csv`로 중간 저장하여 데이터 손실 방지
- **요청 간 sleep**: 모든 API 호출 사이에 1.2초 대기로 rate limit 회피
- **실패 매치 스킵**: `None` 반환 시 카운트만 증가시키고 진행 지속

## 6. 결론

### 6.1 핵심 발견사항

1. **초반 15분 데이터만으로 최대 정확도 78.6%, ROC-AUC 86.5% 달성**
   게임 결과의 약 4/5는 15분 시점에 이미 결정되며, 이는 게임 분석/전략 수립에 충분히 활용 가능한 수준이다.

2. **정글이 가장 중요한 라인**
   - 골드 격차 상관계수 1위 (0.3940)
   - KDA 상관계수 1위 (0.3260)
   - Feature Importance 1위 (`jungle_gold_diff`, 0.1593)
   - 골드 하위 25%일 때 승률 최저(23.88%)
   - 모든 분석에서 일관되게 정글이 최상위로 나타났다.

3. **원딜은 CS가 가장 중요한 유일한 라인**
   - CS 격차 상관계수 1위 (0.3066)
   - Feature Importance 2위 (`bottom_gold_diff`, 0.1217)
   - 원딜 CS 격차 20개로 승률 41%p 변동

4. **서폿은 일반적 지표로 평가가 어려운 포지션**
   - 모든 개인 상관계수가 다른 라인 대비 가장 낮음
   - 특히 CS 상관계수가 음수(-0.0826)로, 자원 분배 차원에서 다른 라인이 CS를 챙기는 게 유리하다는 신호

5. **상대 격차(diff) 피처가 절대 수치보다 예측력 우수**
   - Feature Importance 상위 6개 중 5개가 `*_gold_diff`
   - V2 피처셋에서 라인 격차 추가로 모든 모델 성능 향상 (RF: +3.6%p)

6. **모델 안정성 확보**
   - 모든 모델의 단순분할 vs CV 차이 최대 1.6%p 이내 → 과적합 없음
   - 5-fold CV 표준편차 ±0.5~1.2% → 안정적

### 6.2 한계점

1. **데이터 편향**: 상위 티어(Diamond I 이상)만 수집하여 저티어 / 일반 게임의 패턴은 반영되지 않음
2. **챔피언 정보 미활용**: 167개 챔피언, 6개 역할군, 6개 조합 카테고리를 인코딩 없이 삭제하여 메타 변화나 챔피언 시너지 효과 미반영
3. **시계열 정보 손실**: 15분 시점의 스냅샷만 사용 — 0~15분 사이의 추이(예: 골드 격차가 벌어진 시점, 첫 한타 시기) 정보 누락
4. **수집 실패율 27.5%**: 항복으로 조기 종료된 게임이 다수 누락 — 일방적인 게임 패턴 학습 부족 가능성
5. **단일 시점 모델**: 5분/10분/20분 등 시점별 비교 미수행

### 6.3 개선 방향

1. **챔피언/조합 정보 활용**: 챔피언명을 임베딩 또는 원-핫 인코딩으로 처리하여 메타 변수 추가
2. **다중 시점 분석**: 5분, 10분, 15분, 20분 등 다양한 시점에서 모델 학습 후 비교
3. **시계열 피처 추가**: 골드 격차의 누적 추이, 첫 오브젝트 획득 시간 등 시간 동학 반영
4. **다중 티어 데이터**: 브론즈~플래티넘 데이터 추가 수집 후 티어별 모델 또는 통합 모델 구축
5. **앙상블 기법**: V2_LR과 튜닝된 XGBoost를 결합한 앙상블로 추가 성능 향상 시도
6. **실시간 예측 시스템**: 학습된 V2_LR / `xgb_tuned.pkl` 모델을 활용한 라이브 게임 승패 확률 예측 서비스 구축
